# Live Memory

Esta carpeta es para la memoria viva diaria. No es la fuente formal ni el "código fuente" del negocio, sino el buffer operativo del proyecto.

## Subcarpetas (crear según necesidad)
- `daily/`: Para notas del día, Sprints o logs operativos.
- `sessions/`: Resúmenes de conversaciones de los agentes.
- `inbox/`: Elementos pendientes por procesar.

## Regla de Oro
Todo lo que se estabilice en la memoria viva o se vuelva importante (como una decisión de arquitectura o modelo de datos maduro), debe migrarse hacia `current_state.md` o las carpetas de `foundations/` y `decisions/`.
